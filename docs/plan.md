# Fairtoon 프리뷰 고도화 기획서


작성일: 2026-06-16
승인: 최지호님 (2026-06-16)


## 목적

기존 `Fairtoon-Preview_6.html`(단일 파일 SPA)의 **전체 디자인과 사이트 구조를 유지**하면서, 프리뷰 자체의 완성도를 높인다. 이번 작업은 **프리뷰 품질 향상**이며, 실제 사이트 구축이 아니다.


## 범위에서 제외 (이번 작업 아님 — 추후 실제 구축 단계)

- 데이터베이스 규모 확장 (12편 샘플 유지, 150~300편 확장 안 함)
- 실제 표지 이미지 적용 (그라데이션 플레이스홀더 유지)
- 실제 기사 작성 / 실제 플랫폼 listing 데이터
- 실데이터 기반 전체 한국어 콘텐츠 번역
- Explore 페이지 (당분간 미개발 명시)


## 디자인 원칙

- 레퍼런스 스크린샷의 **룩&필만** 차용: 밝은 테마, Archivo Black 디스플레이 + Inter 본문, 블루(#2F5FEF) 1단어 액센트, pill 내비/버튼, 깔끔한 검색바, EN/언어 토글.
- 스크린샷의 캠페인성 히어로 문구·페이지 구조는 따르지 않음.
- Home·검색결과는 순수 독자 유틸리티 유지. 계몽성 메시지는 Articles/Stay Safe에만.


## 실행 단계


### Phase 1 — 비주얼·디자인 정제

- 타이포 스케일, 여백, 히어로 밸런스, 섹션 리듬 정리
- 카드/호버/프레스 상태, 그림자, 뷰 전환 트랜지션 폴리시
- 모바일·반응형 정리, 접근성(포커스 상태, 라벨) 패스
- 검증: 데스크탑/모바일 폭에서 레이아웃 깨짐 없음, 키보드 포커스 이동 정상


### Phase 2 — 인터랙션 프로토타입 심화 (기존 샘플 데이터 기반)

- 검색바 실시간 자동완성 드롭다운
- 결과 필터/정렬 UI 데모 (장르 / 플랫폼 / 상태 / 가격모델) — 12개 샘플에서 동작
- 부드러운 뷰 라우팅 + empty/loading 상태
- 검증: 입력 시 자동완성 노출, 필터 적용 시 결과 즉시 갱신


### Phase 3 — 기존 섹션 완성도 (신규 실콘텐츠 없음)

- 모든 샘플 아티클 카드가 리더를 열도록 (본문 없는 것은 "sample" 명시 플레이스홀더)
- 상세페이지 빈틈 보강해 레이아웃 완결성 확보
- 기획서의 미완성 "Website Structure"가 허용하는 적절한 구조 요소 추가 (예: 홈의 "How Fairtoon works" 스트립, 가격모델 범례) — 전부 샘플/플레이스홀더 카피
- 검증: 8개 아티클 전부 클릭 시 리더 진입, 깨진 빈 영역 없음


### Phase 4 — 언어 토글 데모화

- EN/KO 토글이 사이트 전반의 UI 라벨/크롬에 반영 (샘플 아티클·시리즈 본문 번역은 제외 — 추후 실콘텐츠 작업)
- 검증: 토글 전환 시 내비·섹션 라벨·버튼·히어로 카피가 언어에 맞게 변경
